# mp-prd.md — 微信小程序里程碑 PRD + 技术选型

> 本文件只覆盖「新增微信小程序」这件事。产品整体卖点见 [README.md](../../README.md)。

## 1. 要做什么

把现有网页版 SoulTranslator 的核心体验，搬到**微信小程序**上（一份小程序同时跑 iOS / Android 微信），并通过统一框架让网页和小程序共用一套前端代码。后端复用现有 Express 代理。

## 2. 给谁

- 现有网页用户里、习惯在微信内使用、不愿跳浏览器的人群。
- 微信生态内的新增流量（搜一搜、好友转发卡片、分享长图带来的拉新）。

## 3. MVP 范围（第一版只做这些）

✅ **做：**
- 核心分析：四象限社交场域选择 + 关系亲密度滑动条 + 文字输入 + 截图识别（OCR）→ 输出 4 套嘴替卡片 + 关系胜率红绿灯。
- 复制嘴替到剪贴板。
- 分享 / 转发：微信原生转发卡片 + 生成分享长图（替代网页的 html2canvas）。

❌ **明确不做（留后续版本）：**
- 微信登录 + 历史记录（要后端加数据库，MVP 不碰）。
- 付费 / 会员（要支付能力，放后续）。
- 网页旧 HTML 的彻底下线（先并存，小程序稳定后再考虑用 uni-app 的 H5 产物替换网页）。

## 4. 成功标准

1. 微信开发者工具里能完整跑通：选场域 → 拖滑块 → 输入/传图 → 出 4 卡片 → 复制 → 转发。
2. 同一套 uni-app 代码 `npm run dev:h5`（或 HBuilderX 运行到浏览器）也能跑出等价网页。
3. 真机（iOS + Android 各一台）体验版功能正常。
4. 通过微信审核并发布（依赖：企业/个体主体 + 域名备案 + 内容安全接入）。

## 5. 关键约束 / 风险

- **域名备案**：`wx.request` 合法域名必须已 ICP 备案。开发期用「不校验域名」绕过；上线前必须就绪。备案/迁国内服务器并行进行，不阻塞开发。
- **内容审核**：情感 / AI 生成内容类，需接入微信内容安全接口（msgSecCheck / imgSecCheck）过滤输入与输出，否则大概率审核被拒。
- **主体资质**：用企业 / 个体工商户主体注册（已定），便于选类目、过审。
- **Render 冷启动**：免费套餐休眠，首个请求慢；开发期可接受，上线迁国内解决。

## 6. 技术选型（最简但稳健，每项说明为什么 / 否决了什么）

| 选型 | 决定 | 为什么 / 否决了什么 |
|---|---|---|
| 前端框架 | **uni-app（Vue3 + Vite）** | 一套代码出微信小程序 + H5；Vue 语法对新手友好；中文文档/社区最全。**否决** Taro（React，新手门槛略高）、微信原生 WXML（两套代码维护痛）、Flutter/RN（出的是原生 App，进不了微信小程序，过重）。 |
| 开发工具 | **VS Code + uni-app CLI（Vite）为主** | 与 Claude Code 协作流一致（终端可驱动：建项目/编译/改码）。**否决** HBuilderX（图形界面，AI 无法驱动；且会多装一个独立 IDE）。预览小程序仍用微信开发者工具。 |
| 样式 | 普通 CSS/SCSS + rpx 响应式 | MVP 先用普通样式还原。**否决** 直接搬 Tailwind（小程序需 weapp-tailwindcss 额外工具链，新手期徒增复杂度，后续可选上）。 |
| 后端 | **复用现有 Express（Render）** | 零重写，网页/小程序共用。**否决** 微信云开发（绑死微信生态、网页端用不了、要迁移现有逻辑）。 |
| 网络请求 | 封装统一 `request` 模块（条件编译） | 小程序用 `uni.request`，H5 用 `uni.request`/fetch；屏蔽差异。 |
| 截图转 base64 | 条件编译 | H5 用 `FileReader`；小程序用 `uni.getFileSystemManager().readFile({encoding:'base64'})`。 |
| 分享长图 | `<canvas type="2d">` + `uni.canvasToTempFilePath` | 替代网页的 html2canvas（小程序无 DOM，html2canvas 不可用）。 |
| 内容安全 | 后端新增 `/api/sec-check`（调微信 msgSecCheck/imgSecCheck） | 上线合规必需，集中在后端做。 |
